# 컴퓨터공학 및 정보보안 용어 정리

## 발견된 용어 (총 20개)


### 보안
**빈도**: 35회
**설명**: 컴퓨터 시스템과 정보를 비인가 접근이나 손상으로부터 보호하는 것


### 알고리즘
**빈도**: 19회
**설명**: 문제 해결을 위해 컴퓨터가 수행할 일련의 단계적 절차


### 해킹
**빈도**: 16회
**설명**: 컴퓨터 시스템에 비인가 접근을 시도하는 행위


### 취약점
**빈도**: 11회
**설명**: 보안 시스템의 약점으로 악용될 수 있는 부분


### 프로그래밍
**빈도**: 10회
**설명**: 컴퓨터에게 특정 작업을 수행하도록 지시하는 명령어들을 작성하는 과정


### 암호
**빈도**: 9회
**설명**: 정보를 비밀로 유지하기 위해 사용되는 코드화된 메시지


### 웹
**빈도**: 9회
**설명**: 인터넷을 통해 정보를 공유하는 시스템


### 서버
**빈도**: 4회
**설명**: 다른 컴퓨터(클라이언트)에 서비스를 제공하는 컴퓨터


### 네트워크
**빈도**: 3회
**설명**: 컴퓨터들을 상호연결하여 정보를 교환하는 시스템


### 인증
**빈도**: 3회
**설명**: 사용자의 신원을 확인하는 프로세스


### 데이터베이스
**빈도**: 2회
**설명**: 체계적으로 정리하고 관리되는 데이터의 집합


### 메모리
**빈도**: 2회
**설명**: 컴퓨터가 데이터와 명령어를 임시로 저장하는 장치


### 스크립트
**빈도**: 2회
**설명**: 간단한 자동화 작업을 수행하는 코드


### 방화벽
**빈도**: 1회
**설명**: 승인되지 않은 네트워크 접근을 차단하는 보안 시스템


### 운영체제
**빈도**: 1회
**설명**: 컴퓨터의 하드웨어 자원을 관리하고 소프트웨어가 실행되도록 하는 시스템 소프트웨어


### 프로세스
**빈도**: 1회
**설명**: 실행 중인 프로그램의 인스턴스


### API
**빈도**: 1회
**설명**: 소프트웨어가 서로 통신하는 방식을 정의하는 인터페이스


### 버그
**빈도**: 1회
**설명**: 프로그램의 오류나 결함


### 디버깅
**빈도**: 1회
**설명**: 프로그램의 버그를 찾아 수정하는 과정


### 배포
**빈도**: 1회
**설명**: 완성된 소프트웨어를 사용자에게 제공하는 과정
